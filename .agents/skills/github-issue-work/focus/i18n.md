# Translations (i18n)

Pass to **frontend workers**, **i18n reviewer**, and **fix workers** when UI strings change.

**Full skill:** [frontend-i18n](../../frontend-i18n/SKILL.md) — read before auditing or fixing XLF.

## When this applies

| Change                                        | Action                                                         |
| --------------------------------------------- | -------------------------------------------------------------- |
| New/changed UI copy in templates or TS        | Mark with `i18n` / `$localize`, run extract, update locale XLFs |
| Backend-only issue                            | Skip i18n review                                               |
| New shared ethereal-ui component with text    | Same as frontend                                               |

## Worker workflow (frontend slice)

From `frontend/`:

```bash
bun run extract-i18n
# Then edit src/locale/messages.{locale}.xlf — add <target> for new trans-units
bun run build   # verify all locales compile
```

Translate new units: edit `src/locale/messages.{locale}.xlf` `<target>` per
trans-unit `id` from `messages.xlf`.

Report in deliverable: build outcome; locales still missing targets (if any).

## Verification (reviewer / orchestrator)

```bash
cd frontend
bun run extract-i18n
bun run build
```

### Must-fix (review fails)

- Missing `<target>` in any locale file for a trans-unit present in `messages.xlf`
- Stale translation where English `<source>` changed but `<target>` was not updated
- Build failure due to `i18nMissingTranslation: error`

### Reviewer deliverable

1. Confirm `bun run build` succeeds (all locales)
2. List any locale files with missing or stale targets
3. Pass / fail recommendation

## Fixing translation findings

1. Read [frontend-i18n/SKILL.md](../../frontend-i18n/SKILL.md)
2. Update `messages.{locale}.xlf` targets
3. Re-run `bun run build` until clean

Route fixes to **frontend worker** (or fix worker with `focus/i18n.md` attached).
