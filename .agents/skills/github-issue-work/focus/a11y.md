# Accessibility (a11y)

Pass to **frontend workers**, **maintainability reviewer**, and **Bugbot** (when UI changes).

Standards: `.github/copilot-instructions.md` § Accessibility, `docs/architecture/accessibility.md`.

Target: **WCAG 2.2 AA** for new and changed UI.

## Component rules

- Use `ui-*` primitives (`ui-form-field`, `ui-button`, `ui-input`, …) — they encode labels and errors
- Map `Problem.errors[]` → field `[error]` with `aria-describedby` / live region per `ui-form-field`
- Dialogs/menus: CDK focus trap, restore focus, `role="dialog"` + `aria-modal="true"`
- Route changes: focus moves to `main` heading; skip link present on shell pages

## Reviewer checks

- [ ] Every input has an associated visible label (not placeholder-only)
- [ ] Error messages exposed to assistive tech (`aria-describedby`, `aria-live` where needed)
- [ ] Keyboard: tab order logical; custom widgets operable without pointer
- [ ] Focus visible on interactive elements (design tokens, not removed outlines)
- [ ] Images: informative `alt`; decorative `alt=""`
- [ ] Color not sole indicator of state (icons/text accompany color)
- [ ] Touch targets ≥ 44×44px on primary actions
- [ ] Loading/error states announced or visible (not silent failure)

## Tests

- Colocated specs assert behavior via DOM (Testing Library), not internals
- Critical flows: consider axe-core in spec or Storybook a11y addon when adding shared components
